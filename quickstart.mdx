## Comenzar en tres pasos

Sigue esta guía para obtener tu token y hacer tu primera petición a la API.

### Paso 1: Obtener tu token de autenticación

<AccordionGroup>
  <Accordion icon="key" title="Registrarse e iniciar sesión">
    Para usar la API, necesitas un token de autenticación válido.
    
    1. Visita [https://kontaiapi.com](https://kontaiapi.com)
    2. Crea una cuenta o inicia sesión si ya tienes una
    3. Navega a la sección de API o tokens
    4. Genera un nuevo token de API
    
    <Warning>
      Guarda tu token de forma segura. No lo compartas públicamente ni lo incluyas en código que vayas a versionar.
    </Warning>
  </Accordion>
  
  <Accordion icon="shield" title="Usar el token en tus peticiones">
    Una vez que tengas tu token, inclúyelo en el header `Authorization` de todas tus peticiones:
    
    ```bash
    Authorization: Bearer TU_TOKEN_AQUI
    ```
    
    O en formato curl:
    
    ```bash
    curl -H "Authorization: Bearer TU_TOKEN_AQUI" \
         https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/health
    ```
  </Accordion>
</AccordionGroup>

### Paso 2: Verificar que la API esté funcionando

Una vez que tengas tu token, verifica que la API esté funcionando correctamente:

```bash
curl -H "Authorization: Bearer TU_TOKEN_AQUI" \
     https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/health
```

<Accordion icon="check-circle" title="Respuesta esperada">
  Deberías recibir una respuesta como esta:
  
  ```json
  {
    "message": "API is running",
    "timestamp": "2024-01-01T00:00:00.000Z",
    "service": "api-sii"
  }
  ```
</Accordion>

### Paso 3: Hacer tu primera petición útil

Ahora que tienes tu token y verificaste que la API funciona, puedes probar los endpoints del SII.

<AccordionGroup>
  <Accordion icon="shield-check" title="Verificar estado del login SII">
    Verifica si tus credenciales del SII están funcionando:
    
    ```bash
    curl -H "Authorization: Bearer TU_TOKEN_AQUI" \
         https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/sii/health
    ```
    
    <Info>
      Este endpoint puede tardar entre 10-30 segundos ya que realiza un login completo en el portal del SII.
    </Info>
  </Accordion>
  
  <Accordion icon="file-invoice" title="Emitir una boleta de honorarios">
    Emite una boleta de honorarios automáticamente:
    
    ```bash
    curl -X POST \
         -H "Authorization: Bearer TU_TOKEN_AQUI" \
         -H "Content-Type: application/json" \
         -d '{
           "tipoRetencion": "RETRECEPTOR",
           "rutDestinatario": "12345678-9",
           "descripcionPrestacion": "Servicios de consultoría",
           "valorPrestacion": 500000
         }' \
         https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/sii/boleta
    ```
    
    <Warning>
      Este endpoint puede tardar entre 60-120 segundos ya que realiza todo el proceso de emisión de boleta de forma automatizada.
    </Warning>
  </Accordion>
</AccordionGroup>

## Ejemplos en diferentes lenguajes

<CardGroup cols={2}>

<Card title="JavaScript/Node.js" icon="square-js">
```javascript
const response = await fetch(
  'https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/health',
  {
    headers: {
      'Authorization': 'Bearer TU_TOKEN_AQUI'
    }
  }
);
const data = await response.json();
console.log(data);
```
</Card>

<Card title="Python" icon="python">
```python
import requests

headers = {
    'Authorization': 'Bearer TU_TOKEN_AQUI'
}

response = requests.get(
    'https://ql3xbjhoe0.execute-api.us-east-1.amazonaws.com/dev/health',
    headers=headers
)
print(response.json())
```
</Card>
